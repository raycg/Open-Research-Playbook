# Research Playbook

> **How do we conduct research?**

This document defines the standard research workflow used in the laboratory.

---

## Why?

Good research should be:

* Planned before action.
* Verified with evidence.
* Documented clearly.
* Transferable to the next researcher.

---

## What?

### Research Lifecycle

| Lifecycle       | Daily Practice                                                                                                                             | Output                 |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------- |
| 🧠 **Think**    | [Daily Note](../templates/G-daily-note.md) (Morning Plan)                                                                                  | Daily Plan             |
| 🔬 **Verify**   | [Daily Note](../templates/G-daily-note.md) (Evening Report) + [AI Daily Self-review Prompt](../templates/E-ai-daily-self-review-prompt.md) | Daily Report           |
| 📝 **Document** | [Meeting Notes](../templates/F-meeting-notes.md)                                                                                           | Reproducible Knowledge |
| 🤝 **Transfer** | [Knowledge Transfer](10-knowledge-transfer.md)                                                                                             | Research Continuity    |

---

### 🧠 Think

Before starting work, prepare a **[Daily Note](../templates/G-daily-note.md)**.

The morning plan should answer:

* What is today's objective?
* How does it relate to my short-term goal?
* What evidence will be collected?
* What result is expected?
* What risks may occur?

---

### 🔬 Verify

Before leaving, complete the **Evening Report** in the **[Daily Note](../templates/G-daily-note.md)**.

Then review it using the **[AI Daily Self-review Prompt](../templates/E-ai-daily-self-review-prompt.md)**.

Compare:

* Morning plan
* Actual work
* Evidence
* Results

---

### 📝 Document

Record research knowledge using:

* **[Meeting Notes](../templates/F-meeting-notes.md)**
* **[Knowledge Transfer Checklist](../templates/B-knowledge-transfer-checklist.md)** (when preparing for project transfer)

Documentation should preserve:

* Design decisions
* Experiment procedures
* Error logs
* Reproducibility notes

---

### 🤝 Transfer

Prepare research knowledge using the **[Knowledge Transfer](10-knowledge-transfer.md)** workflow.

Knowledge transfer is supported by:

* **[Knowledge Transfer Checklist](../templates/B-knowledge-transfer-checklist.md)**
* **[Verification Report](../templates/C-verification-report.md)**
* **[Acceptance Form](../templates/D-acceptance-form.md)**
